Rotas

  • GET /usuarios
    • Restrição: Apenas para administradores.
    • Retorno: Lista de usuários em formato JSON.
    • Exemplo de retorno:
      • [
  {
    "id": 1,
    "nome": "João da Silva",
    "nick": "Silva+10",
    "email": "joao.silva@example.com"
  },
  {
    "id": 2,
    "nome": "Maria Souza",
    "nick": "m_Silva",
    "email": "maria.souza@example.com"
  }
]
  • GET /usuarios/<id>
    • Restrição: Administradores ou usuário logado com mesmo <id> solicitado.
    • Retorno: Detalhes de um usuário específico em JSON.
    • Exemplo de retorno:
      • {
  "id": 1,
  "nome": "João da Silva",
  "nick": "Silva+10",
  "email": "joao.silva@example.com"
}
  • POST /usuarios
    • Restrição: Nenhuma.
    • Campos enviados: Devem ser enviados via formulário.
    • Campos do formulário:
      • nome: Nome do usuário
      • email: Email do usuário
      • nick: Apelido do usuário (opcional)
      • senha: Senha do usuário
      • senha_confirmacao: Confirmação da senha do usuário
    • Retorno: JSON com os dados do usuário criado.
  • PUT /usuarios/<id>
    • Restrição: Administradores ou usuário logado com mesmo <id> solicitado.
    • Campos enviados: Devem ser enviados via formulário.
    • Campos do formulário:
      • nome: Nome atualizado
      • email: Email atualizado
      • nick: Apelido atualizado (opcional)
      • senha_atual: Senha atual (opcional*)
      • senha_nova: Nova senha (opcional*)
      • senha_confirmacao: Confirmação da nova senha (opcional*)
        • *Caso não deseje atualiar a senha.
    • Retorno: JSON com os dados atualizados do usuário.
  • DELETE /usuarios/<id>
    • Restrição: Apenas para administradores.
    • Retorno: JSON confirmando a exclusão.
