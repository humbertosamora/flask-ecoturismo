Observações Gerais

  • O campo id_usuario representa o identificador do usuário que criou o registro.
  • Os campos iniciados por "flag" indicam se a informação se aplica a um tópico específico descrito pelo flag (por exemplo, praia, montanha, etc.).
  • Um usuário só pode alterar ou excluir as dicas cadastradas por si mesmo. O administrador, no entanto, possui controle total sobre qualquer dica.

  • GET /dicas/<id>
    • Restrição: Nenhuma.
    • Retorno: Detalhes de uma dica específica em JSON.
    • Exemplo de retorno:
      • {
  "id": 1,
  "id_usuario": 3,
  "titulo": "Trilha Segura",

  "flag_praia": False,

  "flag_cachoeira": True,

  • POST /dicas
    • Restrição: Apenas para usuários autenticados.
    • Campos do formulário:
      • titulo: Título da dica (obrigatório)
      • descricao: Descrição detalhada da dica (obrigatório)
      • lugar: Local relacionado à dica (caso seja específica)
      • link: Link para mais informações
      • flag_praia: True/False
      • flag_montanha: True/False
      • flag_cachoeira: True/False
      • flag_camping: True/False
      • flag_vestuario: True/False
      • flag_alimentacao: True/False
    • Retorno: JSON com os dados da dica criada.
  • PUT /dicas/<id>
    • Restrição: Apenas para usuários autenticados.
    • Campos do formulário:
      • titulo: Título atualizado (obrigatório)
      • descricao: Descrição atualizada (obrigatório)
      • lugar: Local atualizado
      • link: Link atualizado
      • flag_praia: True/False
      • flag_montanha: True/False
      • flag_cachoeira: True/False
      • flag_camping: True/False
      • flag_vestuari True/False
      • flag_alimentaca True/False
    • Retorno: JSON com os dados atualizados da dica.
  • DELETE /dicas/<id>
    • Restrição: Apenas para usuários autenticados.
    • Retorno: JSON confirmando a exclusão.
